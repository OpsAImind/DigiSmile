DigiSmile – Washington, D.C. SEO Implementation Plan

Owner: Web/SEO Dev
Date: 2025-10-11
Scope: Create dedicated D.C. service pages and a blog to increase local organic traffic and conversions.

1) Objectives
- Create separate SEO-focused pages for each target service in Washington, D.C.
- Optimize on-page SEO (titles, H1, meta descriptions, canonical, OpenGraph, JSON-LD).
- Establish a lightweight blog (index + posts) to publish educational content.
- Integrate clear CTAs driving to Appointment flow.

3) Build Approach in This Codebase
- Framework: Next.js App Router (src/app/...).
- Reuse: `src/features/services/ClinicServiceDetail.tsx`, `src/projectContants/index.tsx` for assets and patterns.
- Create a reusable ServiceDetail template component (if the existing one is insufficient) with props:
  - title, intro, longDescription, benefits (list), procedureSteps (list), FAQs (list), images (array), videoUrl?, locationCTA.
- Create one page file per URL with hardcoded/structured content for SEO uniqueness (avoid a fully dynamic generic page for SEO differentiation).
- Add CTAs that route to `src/app/appointment/page.tsx` or a specific appointment step.

5) SEO Requirements per Page
- Title: "<Service> in Washington, D.C. | Smile Experts Dental"
- Meta Description: 150–160 chars, include service + Washington, D.C.
- H1: "<Service> in Washington, D.C."
- Canonical: absolute URL to production domain + page path
- OpenGraph/Twitter: og:title, og:description, og:url, og:image
- JSON-LD: Service schema with areaServed Washington, D.C. and provider Org
- Breadcrumbs JSON-LD
- Internal links: link to related services and /appointment, /locations if exists
- Image alt text including service and location where natural

6) Content Structure per Page (minimum)
- Hero section: H1, short intro, primary CTA "Book Appointment"
- What is <Service>?
- Benefits/Why Choose Us in D.C. (3–6 bullets)
- Procedure/What to Expect (steps)
- Who is a Candidate? (optional)
- Before & After or Gallery (reuse `src/features/smile-gallery` if applicable)
- Pricing/Insurance (if allowed; avoid precise pricing if policy forbids)
- FAQs (3–6; leverage/extend `projectContants.faqs` if relevant)
- Strong CTA section with phone and booking link

7) Blog Setup
- Routes:
  - src/app/blog/page.tsx (index with list of posts)
  - src/app/blog/[slug]/page.tsx (post detail)
- Content Source:
  - Option A: MDX-based posts in `content/blog/*.mdx`
  - Option B: TSX-based posts with a small registry (faster to ship initially)
- SEO: titles, meta, OpenGraph, canonical; Article JSON-LD on post pages
- Categories/Tags (optional v1): static filter chips

8) Navigation & Footer
- Update `src/components/Layout/Navbar.tsx` and `Footer.tsx` to include links to the new service pages and /blog.
- Consider a dropdown under "Services" with the 9 new D.C. pages.

9) Sitemap & Robots
- Update `next-sitemap.config.js` to ensure the new routes are included (auto picks all static routes).
- Confirm `npm run postbuild` generates sitemap after build.
- Ensure robots.txt allows crawling; noindex any staging if necessary.

10) Tracking & Conversions
- Ensure CTAs point to `/appointment` with UTM parameters (optional) for tracking.
- Add phone number tel: links with proper click tracking (if analytics is present).

11) Performance & UX
- Use optimized images via `next/image`.
- Lazy-load non-critical media.
- Mobile-first layout; verify with Chakra/MUI responsive props.

12) Acceptance Criteria
- 9 new service pages live under the specified URLs with unique, D.C.-focused content.
- Each page has proper SEO metadata, OpenGraph, canonical, JSON-LD (Service + BreadcrumbList).
- Navbar and footer link to the pages.
- Blog index and one example post are accessible; posts render with Article JSON-LD.
- Pages link to appointment flow and load on mobile and desktop without layout issues.
- Sitemap contains the pages after build.
